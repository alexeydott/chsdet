# Tests

This directory contains the DUnitX unit tests and the reproducible corpus used to validate `chsdet`.

## Layout

- `ChsDet.Tests.pas`: main DUnitX fixture for wrapper behavior and selected encoding regressions
- `ChsDet.Corpus.pas`: helper for loading corpus cases and building benchmark results
- `corpus/manifest.tsv`: source of truth for corpus expectations
- `corpus/data/`: generated byte fixtures consumed by tests and the benchmark
- `corpus/generate-fixtures.ps1`: regenerates `corpus/data`
- `corpus/MATRIX.md`: latest corpus benchmark matrix

## Running Tests

Use the RAD Studio environment before invoking the Delphi compilers:

```powershell
cmd /d /s /c '"D:\Embarcadero RAD Studio\23.0\bin\rsvars.bat" && dcc64 -B -Ebin64 -N0bin64 -Itests ChsDetTests.dpr && bin64\ChsDetTests.exe --hidebanner --consolemode:Quiet'
```

Win32 smoke test:

```powershell
cmd /d /s /c '"D:\Embarcadero RAD Studio\23.0\bin\rsvars.bat" && dcc32 -B -Ebin32 -N0bin32 -Itests ChsDetTests.dpr && bin32\ChsDetTests.exe --hidebanner --consolemode:Quiet --run:ChsDet.Tests.TChsDetectorTests.DetectsCp437FromCorpusNfoArt,ChsDet.Tests.TChsDetectorTests.DetectsIbm850FromCorpus,ChsDet.Tests.TChsDetectorTests.DetectsIbm852FromCorpus,ChsDet.Tests.TChsDetectorTests.DetectsIbm858FromCorpus,ChsDet.Tests.TChsDetectorTests.DetectsWindows1250FromCorpus,ChsDet.Tests.TChsDetectorTests.DetectsKoi8UFromCorpus'
```

## Corpus Workflow

Regenerate fixtures after changing `corpus/generate-fixtures.ps1`:

```powershell
powershell -ExecutionPolicy Bypass -File tests\corpus\generate-fixtures.ps1
```

Rebuild and run the corpus benchmark:

```powershell
cmd /d /s /c '"D:\Embarcadero RAD Studio\23.0\bin\rsvars.bat" && dcc64 -B -Ebin64 -N0bin64 -Itests ChsDetCorpusBench.dpr && bin64\ChsDetCorpusBench.exe'
```

The benchmark rewrites `corpus/MATRIX.md`. Strict rows are enforced by `CorpusStrictBenchmarkPasses`; informational rows are tracked for visibility only.

## Legacy Single-Byte Coverage

`corpus/data/cp437-nfo-art.txt` is generated from NFO-style pseudo-graphics encoded as code page 437. It protects the DOS/OEM art heuristic and should detect as:

- Name: `IBM437`
- Code page: `437`
- Language: `DOS`

`corpus/data/cp437-plain-no-art.txt` is generated as a negative regression fixture. It is encoded as CP437 but intentionally has no box/block drawing bytes, and the test suite verifies that it is not reported as `IBM437`.

`corpus/data/cp437-small-nfo-art.txt` covers a smaller mixed text/art NFO sample. The CJK corpus fixtures are also checked explicitly so multibyte legacy text does not false-positive as `IBM437`.

Additional generated fixtures cover `IBM850`, `IBM852`, `IBM858`, `windows-1250`, and `KOI8-U`. The related unit tests use corpus fixtures rather than ad hoc sample files.

`DisableCodePage` is covered for the heuristic-only paths as well: CP437 pseudo-graphics, OEM text signatures, `windows-1250`, and `KOI8-U` must not be reported after their code page has been disabled.
